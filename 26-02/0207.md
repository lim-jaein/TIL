# LLM과 MCP
## LLM
LLM(Large Language Model) 대규모 언어 모델은 대량의 텍스트 데이터로 학습된 머신 러닝 모델
<br><br>

## MCP가 나오게 된 배경

### 1) LLM과 외부 도구 연동의 필요성

LLM을 사용하다보면, 그럴듯한 출력을 하며 정확한 답이 아닌 결과를 내릴 때가 있음 (할루시네이션)

이 때 LLM이 외부의 신뢰할 수 있는 데이터 소스나 도구와 연결되면, 최신 정보 기반 응답, 검증 가능한 답변, 그리고 실제 실행 가능한 결과를 제공할 수 있다.

### 2) API 개별 구현의 복잡성

기존에는 각 데이터 소스마다 다른 API, 인증방식, 데이터 포맷을 개별적으로 구현해야함

MCP는 **표준화된 프로토콜**을 제공하여 다양한 데이터 소스 접근을 통일된 방식으로 처리할 수 있게 됨

## MCP란?

MCP(Model Context Protocol)란 다양한 LLM 기반의 AI 시스템과 외부 데이터 소스 또는 도구를 표준화된 방식으로 연결해주는 개방형 프로토콜이다.
쉽게 말해, USB-C 표준처럼 어떤 환경에서도 동일한 방식으로 도구를 연결하고 사용할 수 있게 해주는 규약입니다.

![image.png](attachment:34be2506-6736-4344-8e8d-133557d8cb5c:image.png)

> MCP는 앤트로픽(Claude 개발사)이 2024년 11월 26일 발표한 개방형 프로토콜이다.

<br>
## MCP 아키텍처 구성 요소

MCP는 세 가지 주요 구성 요소로 이루어진다. 

**호스트(Host), 클라이언트(Client), 그리고 서버(Server)**이다.

```java
┌─────────────────────┐
│   MCP Host          │  (예: Cursor AI, Claude Desktop)
│  ┌───────────────┐  │
│  │  MCP Client   │  │  ← LLM 애플리케이션에 내장된 MCP 통신 모듈
│  └───────┬───────┘  │
└──────────┼──────────┘
           │ MCP Protocol
           ↓
    ┌─────────────┐
    │ MCP Server  │  (예: filesystem, slack, github)
    └─────────────┘
```

| 구성 요소 | 역할 | 설명 |
| --- | --- | --- |
| 호스트(Host) | LLM 앱 | Cursor AI, Claude Desktop 등, AI를 사용하는 애플리케이션 |
| 클라이언트(Client) | 서버와 연결하는 다리 | 호스트 안의 부품으로, 하나의 MCP 서버와 1:1 연결을 담당. MCP 프로토콜을 기반으로 서버에 요청을 보내고, 응답을 받아서 모델에 전달하는 역할 |
| 서버(Server) | 외부 정보를 제공 | 실제 데이터나 기능을 제공하는 서버. 예를 들어 파일 서버는 파일을 읽어서 내용을 제공하고, 날씨 서버는 현재 날씨 데이터를 제공함 |
<br>

## MCP Server가 제공하는 핵심 요소

| 구성 요소 | 역할 | 설명 |
| --- | --- | --- |
| 리소스(Resources) | 데이터 제공 | LLM이 읽을 수 있는 정보나 컨텍스트를 제공 |
| 도구(Tools) | 기능 실행 | LLM이 호출하여 실행할 수 있는 함수 제공 |
| 프롬프트(Prompts) | 템플릿 제공 | 재사용 가능한 프롬프트 템플릿 제공 |
